## Overview

FundSim is a private equity / VC fund simulator built as a React + TypeScript SPA with a minimal Express.js backend. Users can model a fund's full lifecycle тАФ capital deployment, J-curve, distribution waterfall (European or American), and performance metrics (IRR, TVPI, DPI, PME) тАФ entirely in the browser. Authentication is handled server-side via Google OAuth 2.0 and stateless JWTs; all financial computation is client-side.

### Architectural Style

**SPA + Auth-only Serverless API** тАФ All simulation logic runs client-side; the server is a thin authentication layer only.

### Key Design Decisions

- All domain computation is client-side (no server-side simulation endpoints)
- Stateless JWT in httpOnly cookie тАФ no database or session store required
- `useMemo` for all four computed outputs тАФ inputs change triggers full recalculation
- `fundsim.db` exists on disk but `database.js` is an intentional stub ("auth is stateless JWT")
- Demo mode via `localStorage` fallback when server is unreachable

| Carried Interest (Carry) | GP's profit share (typically 20%), earned after hurdle      |

| Harvest Period           | Years after investment period; fund exits investments       |

| American Waterfall       | Carry calculated deal-by-deal                               |

| DPI                      | Distributions to Paid-In тАФ cash returned ├╖ cash called      |

## Key Workflows

1. **Auth (Google OAuth):** User clicks "Continue with Google" тЖТ browser navigates to `/api/auth/google` тЖТ Passport redirects to Google consent тЖТ callback to `/api/auth/google/callback` тЖТ server mints 7-day JWT тЖТ sets `fundsim_token` httpOnly cookie тЖТ redirects to client app

2. **Demo Auth:** User fills email/password form тЖТ client stores `{ name, email }` in `localStorage.fundsim_auth` тЖТ App reads it as valid session (no server call)

3. **Simulation:** User adjusts slider in `GlobalInputs` тЖТ `setInput(key, value)` updates `FundInputs` state тЖТ React invalidates four `useMemo` calls тЖТ `calculateLifecycle/jCurve/waterfall/performance` re-run synchronously тЖТ all four tabs re-render with new values

4. **Waterfall (European):** Return of Capital (LP + GP) тЖТ Preferred Return (LP only, compounded hurdle) тЖТ GP Catch-Up (GP catches to carry %) тЖТ Carried Interest Split (carry % GP / remainder LP)

5. **Waterfall (American):** Same 4 tiers computed independently per deal in `dealMultiples[]`, then aggregated

6. **IRR Calculation:** Newton-Raphson iteration (max 1000 steps, tolerance 0.0001) on cash flow array; returns `null` if convergence fails or derivative < 1e-12
