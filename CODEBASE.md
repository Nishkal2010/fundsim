## Overview

FundSim is a private equity / VC / IB finance simulator built as a React + TypeScript SPA. Users can model fund lifecycles, cap tables, SAFE notes, LBOs, and M&A deals entirely in the browser. Authentication is handled by Supabase (Google OAuth + email/password demo mode). Fund model inputs are persisted to Supabase per user. All financial computation is client-side.

### Architectural Style

**SPA + Auth-only Serverless API** — All simulation logic runs client-side; the server is a thin authentication layer only.

### Key Design Decisions

- All domain computation is client-side (no server-side simulation endpoints)
- Stateless JWT in httpOnly cookie — no database or session store required
- `useMemo` for all four computed outputs — inputs change triggers full recalculation
- `fundsim.db` exists on disk but `database.js` is an intentional stub ("auth is stateless JWT")
- Demo mode via `localStorage` fallback when server is unreachable

## Data Layer

**Auth:** Supabase handles authentication. Google OAuth and email/password demo mode are both supported. Demo users are stored in `localStorage` only; full accounts use Supabase Auth.

**Persistence:** Fund model inputs are persisted to the `fund_models` table in Supabase (columns: `user_id`, `inputs` JSONB). Saves are debounced 1.5s after each input change. Anonymous/demo users get in-memory state only.

## Domain Glossary

| Term                     | Definition                                                  |
| ------------------------ | ----------------------------------------------------------- |
| Fund Size                | Total capital committed by all investors (LPs + GP), in $M  |
| LP (Limited Partner)     | Investors who provide capital but don't manage the fund     |
| GP (General Partner)     | Fund manager; makes investments, charges fees, earns carry  |
| Management Fee           | Annual fee (typically 2%) on committed or invested capital  |
| Carried Interest (Carry) | GP's profit share (typically 20%), earned after hurdle      |
| Hurdle Rate              | Minimum annualized LP return before GP earns carry          |
| Investment Period        | Years of active deal-making (typically 3–5)                 |
| Harvest Period           | Years after investment period; fund exits investments       |
| Capital Call / Drawdown  | GP requests LPs wire their committed capital                |
| J-Curve                  | Negative early returns (fees, no exits) then positive later |
| European Waterfall       | Carry calculated on whole-fund performance                  |
| American Waterfall       | Carry calculated deal-by-deal                               |
| Catch-Up                 | GP receives outsized share to reach carry target            |
| Clawback                 | GP returns excess carry if later deals underperform         |
| DPI                      | Distributions to Paid-In — cash returned ÷ cash called      |
| RVPI                     | Residual Value to Paid-In — unrealized value ÷ cash called  |
| TVPI                     | Total Value to Paid-In — (cash + unrealized) ÷ cash called  |
| IRR                      | Internal Rate of Return — annualized, timing-adjusted       |
| MOIC                     | Multiple on Invested Capital — total value ÷ invested       |
| PME                      | Public Market Equivalent — TVPI vs S&P 500 growth           |
| GP Commitment            | GP's own capital in the fund (typically 1–5%)               |

## Key Workflows

1. **Auth (Google OAuth):** User clicks "Continue with Google" → browser navigates to `/api/auth/google` → Passport redirects to Google consent → callback to `/api/auth/google/callback` → server mints 7-day JWT → sets `fundsim_token` httpOnly cookie → redirects to client app

2. **Demo Auth:** User fills email/password form → client stores `{ name, email }` in `localStorage.fundsim_auth` → App reads it as valid session (no server call)

3. **Simulation:** User adjusts slider in `GlobalInputs` → `setInput(key, value)` updates `FundInputs` state → React invalidates four `useMemo` calls → `calculateLifecycle/jCurve/waterfall/performance` re-run synchronously → all four tabs re-render with new values

4. **Waterfall (European):** Return of Capital (LP + GP) → Preferred Return (LP only, compounded hurdle) → GP Catch-Up (GP catches to carry %) → Carried Interest Split (carry % GP / remainder LP)

5. **Waterfall (American):** Same 4 tiers computed independently per deal in `dealMultiples[]`, then aggregated

6. **IRR Calculation:** Newton-Raphson iteration (max 1000 steps, tolerance 0.0001) on cash flow array; returns `null` if convergence fails or derivative < 1e-12
